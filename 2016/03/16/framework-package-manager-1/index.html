PackageManager框架

大部分情况我们是在Activity中使用getPackageManager方法获取一个ApplicationPackageManager的对象，ApplicationPackageManager实际上是包装了一个IPackageManager.Stub.Proxy的对象
由IPackageManager.Stub.Proxy代理执行PackageManager相关操作，IPackageManager.Stub.Proxy实际代理的是PackageManagerService.

APK文件安装过程的细节

下面的过程执行在Package Manger服务中。

  • 等待；
  • 添加一个包文件到安装进程的队列中；
  • 确定合适的地方来安装包文件；
  • 复制apk文件到一个给定的目录下；
  • 确定应用的UID；
  • 请求installd守护程序进程；
  • 创建应用目录和设置权限；
  • 提取dex代码到缓存目录中；
  • 解析packages.list、system、data和packages.xml的最新状态；
  • 向系统发送广播消息，消息带有安装完成效果的名字Intent.ACTION_PACKAGE_ADDED：如果是更新，会带有新的（Intent.ACTION_PACKAGE_REPLACED）。
