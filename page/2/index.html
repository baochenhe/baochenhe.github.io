Android RecyclerView 使用心得记录

前言

本文主要记录和介绍我自己使用RecyclerView心得，如果你是之前没有接触过RecyclerView，一直都是使用的ListView 和 GridView实现的需求，而碰巧你又刚好在犹豫是否是应该牺牲一部分时间，或者牺牲多少时间，来修改自己的老项目的话，那就太好了，因为本文就是我犹豫后的结果。

我们关注的问题无非就几个：

  1. 是否更高效率，更省内存
  2. 是否更好维护
  3. 是否可以写更少的代码
  4. 更listView 和 GridView 相差多少，我要修改多久…
  5. 这么厉害我怎么才能拥有

接下来一起来看看如何去破除这些疑问吧。

背景介绍

RecyclerView是support-v7包中的新组件，是一个强大的滑动组件，与经典的ListView相比，同样拥有item回收复用的功能，但是直接把viewholder的实现封装起来，用户只要实现自己的viewholder就可以了，该组件会自动帮你回收复用每一个item。它不但变得更精简，也变得更加容易使用，而且更容易组合设计出自己需要的滑动布局。

使用它的理由：

RecyclerView is a more advanced and flexible version of ListView. This widget is a container for large sets of views that can be recycled and scrolled very efficiently. Use the RecyclerView widget when you have lists with elements that change dynamically.
简单说，它是ListView的进化，为了当你需要动态展示一组数据的时候就会需要它。

重它的名字不难看出，他更关注的是View回收。我想之前朋友们应该也看过无数的博客怎么分析ListView的是否复用机制，各种闹腾，现在估计还有小伙伴没整明白的。

从这里开第一个点，是否更省内存，其实字面的解析就能知道了，答案是yes!

关于Android的.so文件你所需要知道的

原文链接

早期的Android系统几乎只支持ARMv5的CPU架构，你知道现在它支持多少种吗？7种！

Android系统目前支持以下七种不同的CPU架构：ARMv5，ARMv7 (从2010年起)，x86 (从2011年起)，MIPS (从2012年起)，ARMv8，MIPS64和x86_64 (从2014年起)，每一种都关联着一个相应的ABI。

应用程序二进制接口（Application Binary Interface）定义了二进制文件（尤其是.so文件）如何运行在相应的系统平台上，从使用的指令集，内存对齐到可用的系统函数库。在Android 系统上，每一个CPU架构对应一个ABI：armeabi，armeabi-v7a，x86，mips，arm64- v8a，mips64，x86_64。

为什么你需要重点关注.so文件

如果项目中使用到了NDK，它将会生成.so文件，因此显然你已经在关注它了。如果只是使用Java语言进行编码，你可能在想不需要关注.so文 件了吧，因为Java是跨平台的。但事实上，即使你在项目中只是使用Java语言，很多情况下，你可能并没有意识到项目中依赖的函数库或者引擎库里面已经 嵌入了.so文件，并依赖于不同的ABI。

例如，项目中使用RenderScript支持库，OpenCV，Unity，android-gif-drawable，SQLCipher等，你都已经在生成的APK文件中包含.so文件了，而你需要关注.so文件。

Android应用支持的ABI取决于APK中位于lib/ABI目录中的.so文件，其中ABI可能是上面说过的七种ABI中的一种。

Android6.0 Marshmallow(棉花糖) 对开发者的影响

Android6.0 Marshmallow(棉花糖)
官方原文：
https://developer.android.com/intl/zh-cn/about/versions/marshmallow/android-6.0.htm

Android 6.0新版本包含了许多对系统功能和API的变化，
而我们作为开发者，也要考虑自己的程序需要做哪些兼容性的改变！
优化为主提升体验 Android 6.0亮点解析
http://tech.sina.com.cn/mobile/n/n/2015-09-30/doc-ifxieyms4231929.shtml

系统功能改变

1. Doze and App Standby：

也就是睡眠和app待机。
从Android6.0开始，Android提供了两种延长电池使用寿命的功能：睡眠和app待机。
当设备没有连接电源，进入Doze模式时，让应用程序处于待机状态以此减少电池的消耗
据说，在nexus5和nexus6上测试，平均续航时间提高了30%。(亲儿子数据，仅供参考)
系统进入睡眠状态后，会隔一段时间处理一下正在挂起任务，随着时间的推移，后面的间隔时间会越来越长，以此减少电量的消耗
